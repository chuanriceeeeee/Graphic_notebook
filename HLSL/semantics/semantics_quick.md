### 第一类：顶点输入语义 (提取器)
**作用阶段：** 从模型网格（显存） -> 顶点着色器 `Vertex Shader`。
**本质：** 告诉输入装配器（IA），你想从 3D 模型的源文件中剥离出哪些数据。

| 语义名称 | 常见数据类型 | 数据含义 | 补充说明 |
| :--- | :--- | :--- | :--- |
| **`POSITION`** | `float4` 或 `float3` | **模型空间坐标** | 必须有。这是物体自身的原始坐标。 |
| **`NORMAL`** | `float3` | **顶点法线** | 计算受光、环境反射必用。 |
| **`TANGENT`** | `float4` | **顶点切线** | 用于法线贴图（Normal Map）计算。`w` 分量通常是 $\pm1$，用来决定副切线（Bitangent）的方向。 |
| **`TEXCOORD0` ~ `TEXCOORD7`** | `float2` (也可 3/4) | **UV 坐标** | `0` 是第一套 UV（最常用），`1` 常用于光照贴图（Lightmap）。 |
| **`COLOR`** | `float4` | **顶点颜色** | 美术在建模软件中刷的颜色。NPR 渲染中极其常用，常把阴影阈值或描边粗细藏在 `COLOR.a`（透明通道）里。 |

### 第二类：系统值语义 (SV_ 系)
**作用阶段：** 贯穿整个管线。
**本质：** `SV` 代表 **System-Value**。带有 `SV_` 前缀的语义是被 GPU 硬件底层特殊关照的 VIP 数据。

| 语义名称 | 出现位置 | 数据含义 | 硬件底层的特殊关照 |
| :--- | :--- | :--- | :--- |
| **`SV_POSITION`** | VS 输出 / FS 接收 | **裁剪空间坐标 / 屏幕坐标** | **核心！** 在 VS 输出时，光栅化硬件拿它去切分三角形、生成像素。在 FS 接收时，它变成像素在屏幕上的确切坐标 $(x, y)$。 |
| **`SV_Target`** | FS 输出 | **渲染目标 (屏幕像素)** | 告诉 GPU：“把计算好的颜色写进屏幕缓冲”。可通过 `SV_Target0`、`SV_Target1` 输出到多张贴图。 |
| **`SV_Depth`** | FS 输出 | **深度缓冲** | 允许强行修改像素的深度值。一般不写，GPU 会自动把 `SV_POSITION` 的 `z` 值写入深度图。 |
| **`SV_InstanceID`** | VS 接收 | **实例 ID** | GPU Instancing 时区分同模型的不同实例。 |
| **`SV_VertexID`** | VS 接收 | **顶点编号 ID** | 当前处理的顶点是这个模型里的第几个顶点。 |
| **`SV_IsFrontFace`**| FS 接收 | **是否为正面** | 布尔值/数值。当前像素属于模型正面为 true，背面为 false。常做双面材质。 |

### 第三类：传递语义 (快递箱)
**作用阶段：** 顶点着色器 `Vertex Shader` -> 片元着色器 `Fragment Shader`。
**本质：** 通知光栅化器分配插值寄存器，进行线性插值。

| 语义名称 | 常见用途 | 限制与规范 |
| :--- | :--- | :--- |
| **`TEXCOORD0` ~ `TEXCOORD15`** | 传递一切自定义插值数据 | **现代 HLSL 的万能容器**。可传递 UV、世界坐标、世界法线、视线方向等任意数据。 |
| **`COLOR0` ~ `COLOR1`** | 传递颜色 | 早期 CG 语言必须用 `COLOR` 传颜色。现代 URP/HLSL 中，**强烈建议把所有 VS 传给 FS 的自定义数据，全用 `TEXCOORD` 打包。** |
